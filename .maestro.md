# Maestro Context: Restaurante Las Palmas POS

## Project Intent
Sustituir el sistema actual por un POS de alta gama, alto rendimiento y estética "Antigravity OS", enfocado en la eficiencia operativa en restaurante y cocina.

## Tech Stack
- Frontend: HTML/JS/Vanilla CSS (Antigravity OS Design).
- Integration: Supabase (Postgres).
- Icons: Lucide.

## Workflow Rules (REQUIRED)
1.  **Safety First**: Always use optional chaining and null checks.
2.  **Visual Excellence**: Maintain premium aesthetics in all components.
3.  **High Density**: Optimize space for kitchen production.
4.  **Memory Sync**: Update `.kilocode/rules/memory-bank/` after significant changes.

## Commands Reference
- `/diagnose`: Analyze code for architectural or logic issues.
- `/fortify`: Add error handling and robustness.
- `/streamline`: Optimize performance and simplify logic.
- `/refine`: Final polish pass for UI and UX.

## Core Patterns
- Split-View Production: Operation (Left) | Information (Right).
- Modal Standard: No footers, Windows-style headers, 32px-48px radius.
- Color Token: `#0cf192` (Primary Active), `#1a1f2e` (Main BG).
